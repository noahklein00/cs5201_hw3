Homework 3 - Euler's Method for ODEs and SIR(D) Model

Due on 2020-02-28 at class time

Purpose

This assignment is intended to introduce vector analysis and numerical methods for solving ordinary differential equations in the context of object-oriented C++, and to provide more practice with the following:

  • templated classes
  • templated functions
  • overloading operators
  • the concept of implementing operations as classes
  • C++ to “wrap” a data type

Background

Almost every aspect of the physical world is in some way governed by differential equations, i.e. a means of describing the relationship between a function and its derivative.
A subset of these differential equations, called ordinary differential equations, involve derivatives with respect to a single variable. These ODEs often have no analytical solution and must be solved with numerical methods. The focus of this assignment is the solving of coupled ODEs, or systems of ODEs composed of mutually interdependent equations.

For example, the following is a system consisting of two coupled ODEs:

dpdt=ap+bqdpdt=ap+bq
dqdt=cp+dqdqdt=cp+dq
Note that the derivatives for each dependent variable are functions of both dependent variables pp and qq (aa, bb, cc, and dd are constants). These systems can be solved with Euler’s method, which you should have learned in CS 3200. Instead of solving for yn+1yn+1 by computing y′y′, you will concurrently solve for each element of y⃗ n+1y→n+1 by applying Euler’s method to each differential equation in the system.

In this assignment you will be responsible for an object-oriented implementation of a set of four coupled differential equations: a slightly-modified SIR Model, which is used by epidemiologists to simulate the progress of an infectious disease. The intuition behind this approach is to model members of a given population as a finite-state machine:

The transitions between states are governed by the properties of the disease itself, e.g. transmission rates, recovery rates, death rates, etc. For the purposes of this assignment these properties will remain constant for a given simulation run.

The system of ODEs you will be simulating is as follows:

dSdt=−βISdSdt=−βIS
dIdt=βIS−νI−δIdIdt=βIS−νI−δI
dRdt=νIdRdt=νI
dDdt=δIdDdt=δI

For clarity, S = susceptible, I = infected, R = recovered, D = deceased. ββ is the infection rate, νν is the recovery rate, and δδ is the death rate. Note that S+I+R+DS+I+R+D must always equal NN, the total population size. Note that the state of the system can be represented as a vector consisting of (in this case) four elements, and that all four of the differential equations listed above are functions of (elements of) that vector.

Program Specification

Since the state of a system of ODEs can be represented by a vector, you will first need to implement a templated vector class allowing the creation of vectors of floats, ints, Penguins, or any other class that a user desires. Your vector must have the following operations:

  • A parameterized constructor accepting a 1-dimensional std::initializer_list
  • A parameterized constructor accepting the size of the vector to create
  • Other appropriate constructors
  • A means of resizing and determining the size of the vector with resize and size functions
  • Appropriate functions such that range-based for loops can be used on your vector
  • Copy constructor/copy assignment operations
  • Element access with the [] operator
  • Addition, subtraction, scalar multiplication, and vector multiplication (dot product)
  • Unary minus
  • An apply function accepting a function and returning a new vector containing the result of the function when called on the elements of the calling object, and in which the function f must have the signature T f(T) (for template type T)
  • Stream operators for input/output of vector data - accepted input should be whitespace-delimited elements, and output should be single-space-delimited elements of the vector

All vector arithmetic operations should return new vectors.

You should not use std::vector or std::valarray to create this class.

You will be also responsible for implementing the forward/explicit Euler method for ordinary differential equations. Since we need to support Euler’s method for both scalars and vectors (or any other type, for that matter), and for the solving of any arbitrary ODE, this should also be templated on the type used to encapsulate the state of the differential equation. This class must include the following:

  • A constructor accepting a ODE callback, an initial state y0y0, and a step size hh - an ODE callback is one that accepts a value of the type used to encapsulate the state of the ODE, and returns a value of the same type
  • Other appropriate constructors/destructors/copy functionality
  • function evaluation operator () to “step” the differential equation, calculating and returning the value of the function for the next discrete value of the independent variable, yn+1yn+1, and allowing a custom step size for that step to optionally be specified, overriding the step size specified in the constructor

Finally, you will write a class that represents the SIRD model described above. This does not need to be templated. This model must support simulations in which the state of the system can be updated with respect to a specified time step (the step size of the Euler method). You are responsible for implementing the following operations:

  • A constructor accepting the initial total population (N0N0), the initial number of infected (I0I0), and a step size (hh), and a vector [β,ν,δ][β,ν,δ], which should default to values provided in the driver instructions
  • Other appropriate constructors/destructors/copy functionality
  • function evaluation operator () to “step” the model, calculating the state of the population for the next moment in time
  • A means of writing the current state of the model to a std::ostream (stream operator), using the format Susceptible: <S>, Infected: <I>, Recovered: <R>, Deceased: <D>

The output functionality described above should allow the state of the system to be plotted with respect to time, for example: This run used an initial population of 100 with a single infected individual, a time step of 0.1 days, and a total time of 200 days.

As always, you are expected to follow the assignment guidelines.

Print the state of the model after every time step to stdout.
